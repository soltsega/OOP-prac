**Yes, exactly!** 

### **Upcasting (Automatic - No Error):**
```java
int → long → float → double
byte → short → int → long → float → double
char → int
```

### **Scanner Upcasting Examples:**
```java
nextByte() accepts "25"    // byte → byte ✅
nextShort() accepts "25"   // short → short ✅  
nextInt() accepts "25"     // int → int ✅
nextLong() accepts "25"    // long → long ✅
nextFloat() accepts "25"   // int → float ✅ (upcast)
nextDouble() accepts "25"  // int → double ✅ (upcast)
```

### **Downcasting (Manual - May Need Cast):**
```java
double d = 25.5;
int i = (int) d;  // Needs explicit cast ✅
// i = 25 (truncated)
```

**Rule:** **Upcasting** = automatic (widening)
**Downcasting** = manual (narrowing)

So yes - you can call it **upcasting** that JVM handles automatically.
